# Kubeflow Community: AI-Assisted/AI-Led Contribution Policy

The use of AI to generate code or review is permitted, but the author/contributor must adhere to the following guidelines.

## 1. Accountability

- **Human-in-the-Loop:** You are the sole owner of your contributions. You are responsible for the correctness, security, and maintenance of any code or documentation generated by AI.
- **DCO Compliance:** All submissions must adhere to the [Developer Certificate of Origin](https://developercertificate.org/). You must ensure the AI tool used does not violate the Apache 2.0 license or third-party intellectual property.

## 2. Transparency

- **Disclosure:** If a contribution is substantially generated or refactored by AI (e.g., Claude, Codex, Cursor), include a note in the PR description or use a footer in the commit message:

  ```
  Generated-by: [Tool Name]
  ```

  or

  ```
  Assisted-by: [Agent Name]
  ```

  or 
  ```
  Co-authored-by: [Agent Name]
  ```

- **Verification:** Do not blindly submit AI outputs. All AI-generated code must be locally built, linted, and tested against the Kubeflow test suite before a PR is opened.
- **Explainability:** You must be able to explain and justify any AI-generated logic during the review process. If you can't explain it, don't submit it.

## 3. Quality Standards

- **No "AI Slop":** Maintainers reserve the right to close PRs that appear to be low-effort, mass-generated AI content that adds technical debt or lacks proper context.
- **Security & Best Practices:** AI may suggest insecure patterns. Contributions must adhere to Kubeflow's security requirements, including RBAC configurations and CNCF security benchmarks.

## 4. Community Etiquette

- **Authentic Communication:** AI should be used for code and technical drafting, not for automated responses to maintainer feedback or community discussions. We value human-to-human collaboration.
- **Reviewer Rights:** Maintainers may request that AI-generated logic be refactored or further explained if the intent is unclear.
